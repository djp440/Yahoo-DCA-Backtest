# 智能定投回测系统 - 产品需求规格说明书 (PRD)

> **版本**: v1.0  
> **日期**: 2026/04/16  
> **目标用户**: 个人专业投资者  
> **开发模式**: 个人独立 + Vibe Coding

---

## 1. 产品概述

### 1.1 产品定位

一款面向专业投资者的**桌面端定投回测软件**，提供机构级别的回测能力，支持全资产类别的定投策略研究与投资组合分析。

### 1.2 核心价值主张

- **专业级回测引擎**: 支持复杂策略、蒙特卡洛模拟、滚动回测等专业功能
- **全资产覆盖**: 股票、ETF、加密货币、外汇、期货、债券统一分析
- **极致用户体验**: 明亮清新的苹果设计风格，流畅动画，响应迅速
- **离线优先**: 本地SQLite缓存，数据一次下载，多次使用

### 1.3 用户画像

| 属性 | 描述 |
|------|------|
| 身份 | 个人专业投资者 |
| 投资经验 | 3年以上 |
| 投资周期 | 3-10年中长期投资 |
| 核心需求 | 验证定投策略、构建投资组合、专业分析 |
| 技术偏好 | 追求工具效率，重视UI/UX体验 |

### 2.2 模块详细说明

#### 2.2.1 数据管理模块

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 数据下载 | 支持从Yahoo Finance下载日K线(OHLCV)、基本面、财报、分红、复权数据 | P0 |
| 数据缓存 | 本地SQLite存储，支持按品种、时间范围查询 | P0 |
| 数据更新 | 启动时自动检测并更新本地数据 | P1 |
| 数据导出 | 支持导出为CSV/JSON格式 | P2 |
| 数据导入 | 支持导入外部CSV数据（用于非标品种） | P2 |

**支持品种类型**: 美股个股、美股ETF、加密货币、港股/A股、外汇、期货/期权、债券/固收类

#### 2.2.2 回测引擎模块

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 单笔回测 | 单个品种 + 单个策略的回测 | P0 |
| 组合回测 | 多品种组合整体回测，支持权重配置 | P0 |
| 策略对比 | 同一品种多个策略并行对比 | P1 |
| 蒙特卡洛模拟 | 基于历史数据的随机路径模拟 | P2 |
| 滚动回测 | 滑动窗口回测验证 | P2 |
| Walk-forward分析 | 参数稳定性验证 | P2 |

**支持的定投策略** (全部都要实现):

| 策略名称 | 说明 |
|----------|------|
| 普通定投 | 固定金额、固定时间间隔 |
| 价值平均策略 | Value Averaging，按目标市值调整投入 |
| 智能定投 | 根据估值（PE/PB）动态调整金额 |
| 均线策略 | 价格低于均线多投，高于少投 |
| 金字塔定投 | 越跌越买，越涨越买 |
| 止盈再投策略 | 达到目标收益后止盈继续定投 |
| 动态平衡策略 | 定期再平衡组合权重 |

**定投频率选项**: 每周（指定星期几）、每两周/双周、每月（指定日期）

#### 2.2.3 图表分析模块

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 专业K线图 | OHLCV数据展示，支持缩放平移 | P0 |
| 净值曲线 | 显示策略净值变化 | P0 |
| 成本曲线 | 显示定投平均成本变化 | P0 |
| 回撤曲线 | 显示动态回撤 | P0 |
| 指标叠加 | 支持技术指标叠加到主图 | P1 |
| 多图联动 | 多图表缩放同步 | P2 |
| 图表导出 | 导出为PNG/SVG | P2 |

**技术指标需求** (基于问卷):

| 类别 | 已选指标 |
|------|----------|
| 趋势指标 | 移动平均线(SMA/EMA/WMA) |
| 动量指标 | 相对强弱指数(RSI) |
| 成交量指标 | 成交量(Volume) |
| 波动率指标 | 平均真实波幅(ATR)、标准差 |

**技术指标使用方式**:
1. 仅作为图表展示，不参与回测逻辑
2. 作为定投时机的判断条件（如RSI<30时加倍定投）

#### 2.2.4 回测指标计算模块

需要计算的全部指标列表:

**基础收益指标**:
- 总收益率、年化收益率、超额收益（相对基准）、复利终值

**风险指标**:
- 最大回撤、最大回撤持续时间、波动率（标准差）、下行波动率、VaR、CVaR

**风险调整收益**:
- 夏普比率、索提诺比率、卡玛比率、信息比率、特雷诺比率、欧米伽比率

**定投相关指标**:
- 投入本金总计、持仓市值、投入次数、平均持仓成本、成本曲线、定投收益率vs一次性投入收益率、盈亏平衡点

#### 2.2.5 报告与历史模块

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| PDF报告导出 | 生成专业回测报告 | P1 |
| JSON导出 | 结构化数据导出 | P2 |
| 回测历史 | 保存所有回测记录，支持搜索筛选 | P1 |

#### 2.2.6 日志模块

统一的日志系统，用于调试、问题排查和运行状态监控。

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 多级别日志 | 支持trace/debug/info/warn/error五个日志级别 | P1 |
| 日志分类 | 按模块分类：数据层、回测引擎、UI层、网络请求 | P1 |
| 持久化存储 | 日志持久化到SQLite，默认保留最近30天日志 | P1 |
| 日志查询 | 前端全局日志面板，支持按级别、时间、模块筛选查询 | P2 |
| 日志导出 | 支持导出日志文件，用于问题排查 | P2 |
| 错误上报 | 捕获全局异常，自动记录错误日志和堆栈信息 | P1 |
| 性能埋点 | 记录关键操作耗时（回测计算、数据加载等），用于性能优化 | P2 |

**日志模块设计要点**:
- Rust后端实现核心日志服务，提供统一的日志写入接口
- 前端TypeScript SDK封装，前端代码通过SDK上报日志
- 支持日志级别动态调整，开发环境显示debug级别，生产环境默认info级别
- 全局异常捕获，所有未处理的错误自动写入日志
- 日志包含：时间戳、级别、模块、内容、堆栈信息（错误时）

## 3. 非功能需求

## 7. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| Yahoo Finance API限制 | 数据获取不稳定 | 实现完善的缓存机制，降低请求频率；考虑备选数据源 |
| 回测计算性能瓶颈 | 大数据量计算慢 | 核心计算用Rust实现，异步处理，支持取消操作 |
| 复杂策略实现难度 | 开发周期延长 | MVP聚焦核心策略，高级策略后续迭代 |
| UI动画性能 | 低端设备卡顿 | 提供动画开关，使用CSS硬件加速，必要时降级 |
